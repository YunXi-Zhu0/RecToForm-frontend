# RecToForm 前端 UI 样式交付文档

## 1. 交付目标

本轮 UI 交付目标是把当前页面收敛成一个轻松、简约、排版合理的单页工作台，并落实用户已确认的最新要求。

本次交付不再追求“科技感工作台”，而是以更自然的办公工具体验为主。

## 2. 最新确认的交付要求

### 2.1 主题要求

- 页面不要科技感
- 页面要轻松、简约
- 视觉表达要克制
- 页面不要显得复杂和冗余

### 2.2 结构要求

- `RecToForm` 作为页头商标放在页面上方
- 页面中间要有一个大的上传框
- 上传框支持拖拽上传
- 上传文件后，在同一块区域要求用户选择处理模式
- 处理中时，对同一块区域做背景虚化
- 虚化层上显示进度
- 进度组件中需要有轮盘

### 2.3 信息组织要求

- 首屏只保留最核心流程
- 结果区、失败项区、导出区后置展示
- 不能在未开始任务前把页面做成多栏信息控制台
- 减少英文和控制台化标签

## 3. 交付范围

本次 UI 交付覆盖：

- 页面整体主题和基础样式
- 页头品牌区
- 中央上传主区域
- 上传后同区域模式选择与模板配置
- 处理中虚化遮罩与轮盘进度
- 任务完成后的结果、失败项、导出后置区域

本次 UI 交付不覆盖：

- 后端 API 协议变更
- 业务状态机改写
- 新增识别能力
- 新增导出协议

## 4. 页面交付标准

### 4.1 页头

必须交付：

- 页面顶部品牌 `RecToForm`
- 简洁的头部排版
- 轻量状态标签

不应交付：

- 大面积说明面板
- 强控制台氛围头图
- 过度装饰性发光与科技背景

### 4.2 中央主区域

必须交付：

- 大上传框
- 拖拽上传支持
- 点击上传支持
- 上传限制信息
- 文件列表
- 上传完成后同区域模式选择
- 模板模式下的同区域模板选择

核心标准：

- 上传前后属于同一块主区域
- 流程连续，不跳散
- 主区域始终是首屏唯一视觉中心

### 4.3 处理中覆盖层

必须交付：

- 背景虚化
- 半透明遮罩
- 轮盘式进度组件
- 当前状态与阶段文案
- 必要的任务进度信息

交付重点：

- 强反馈
- 弱压迫感
- 不做科技控制台式复杂面板

### 4.4 结果区

必须交付：

- 任务完成后再出现
- 模板模式只读预览
- 标准字段模式可编辑表格
- 导出区
- 失败项区

结果区标准：

- 后置展开
- 不影响首屏简洁性
- 保留已有业务能力

## 5. 视觉系统交付标准

### 5.1 配色

当前建议交付方向：

- 浅色暖底
- 柔和绿色或中性色作为强调色
- 边框和阴影轻量
- 提示、错误、成功颜色保持清晰但不过度刺激

### 5.2 字体

当前建议交付方向：

- 品牌标题可稍有辨识度
- 正文以自然、清晰、易读为主
- 避免工业感过强的显示字体大面积铺满

### 5.3 控件

当前建议交付方向：

- 主按钮清晰但不刺眼
- 输入框干净、留白足够
- 标签和状态徽标轻量化
- 模式卡片不做强烈发光

## 6. 交付验收点

本轮 UI 样式交付至少要满足以下验收点：

1. 首屏能一眼看到 `RecToForm` 品牌和大上传框。
2. 上传后，模式选择出现在同一主区域内。
3. 处理中时，原区域进入虚化遮罩态，并显示轮盘进度。
4. 整体视觉已从“科技感”切换为“轻松简约”。
5. 结果、导出、失败项不再在未处理前抢占首屏。
6. 页面排版更收敛，不再显得复杂。

## 7. 当前交付结论

后续如继续迭代页面样式，交付方向必须继续遵守以下结论：

- 保持单主区域流程
- 保持简洁页头
- 保持上传后同区域配置
- 保持处理中虚化覆盖层和轮盘
- 保持轻松、简约、非科技感主题

本文件作为后续 UI 验收和补充实现的基线文档。
