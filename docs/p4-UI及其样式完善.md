# P4 UI 及其样式完善

## 1. 阶段目标

P4 阶段聚焦工作台主页面的 UI 收敛与样式完善，在不改动既有接口契约和核心业务流程的前提下，持续优化以下内容：

- 页面结构更清晰
- 上传主流程更集中
- 模板与文件信息分区更明确
- 上传后的预览反馈更直观
- 整体视觉更轻、更简约

## 2. 截至目前的完成情况

### 2.1 首屏结构重构

已完成：

- 将工作台首屏调整为三栏结构
- 左侧固定为模式与模板选择区
- 中间固定为上传与预演主舞台
- 右侧固定为上传文件清单区
- 页面容器已从居中定宽调整为全屏拉宽展示

这意味着首屏的信息密度虽然提升了，但职责已经更稳定：

- 左侧负责“配置”
- 中间负责“上传与预演”
- 右侧负责“文件检查与移除”

### 2.2 上传交互收敛

已完成：

- 中间区域保留大上传框作为主入口
- 支持拖拽上传和点击上传
- 上传后中间区域不再继续堆叠配置表单
- 上传后自动切换为“表格样式预演”状态
- 预演区顶部增加“返回补充文件”按钮

当前的交互含义是：

- 用户第一次进入页面时，优先做上传
- 文件进入队列后，中心区域改为让用户理解“最终会生成怎样的表格形态”
- 如果要继续补文件，可以通过返回按钮切回上传框

### 2.3 模板与标准字段预演

已完成：

- 模板模式下，中间区域根据当前所选模板渲染虚拟表头
- 首列显式预演为“源文件”
- 标准字段模式下，中间区域改为预演标准字段结构
- 预演区采用表头 + 骨架行的虚拟表格形式，而非纯文本标签

这样处理后，上传后的反馈不再只是“文件已上传”，而是“这批文件将按怎样的结构进入结果表格”。

### 2.4 文件清单侧栏

已完成：

- 将文件列表从上传组件中拆出，独立到右侧栏
- 显示文件名
- 显示文件大小
- 显示文件类型
- 显示本地疑似重复或后端重复状态
- 每个文件提供红色 `×` 移除按钮
- 右侧栏支持一键清空文件列表

这使得上传主入口和文件管理职责分离，避免上传区既是主视觉又承担列表管理，导致结构混乱。

### 2.5 处理中保护

已完成：

- 处理中时，中间主舞台继续保留虚化遮罩和进度层
- 处理中时禁用模式切换
- 处理中时禁用模板切换
- 处理中时禁用右侧文件移除和清空操作

这样可以避免任务已创建后，前端状态仍被用户继续修改，导致界面与任务实际状态错位。

### 2.6 主题与样式方向

目前已完成的视觉方向调整包括：

- 使用浅色暖底而不是深色科技风
- 强化大圆角卡片与低对比边框
- 弱化旧版密集面板堆叠感
- 把上传区、模板区、文件区做出明确层次
- 继续保持结果区和导出区作为后置内容

## 5. 当前验收结论

目前已经满足以下要求：

1. 页面可按左、中、右三个区域理解。
2. 左侧可承载模板选择。
3. 中间保留上传选择框。
4. 右侧集中展示上传文件内容。
5. 文件列表显示文件名和文件大小。
6. 文件列表支持红色 `×` 移除。
7. 重复文件状态会显式展示。
8. 上传后中间区域会切换成模板或标准字段预演态。
9. 中间区域顶部提供返回按钮，允许继续补充文件。
10. 页面已改为更充分利用整屏宽度。
