# P4 UI 及其样式完善

## 1. 阶段目标

P4 阶段聚焦工作台主页面的 UI 收敛与样式完善，在不改动既有接口契约和核心业务流程的前提下，持续优化以下内容：

- 页面结构更清晰
- 上传主流程更集中
- 模板与文件信息分区更明确
- 上传后的预览反馈更直观
- 整体视觉更轻、更简约

## 2. 截至目前的完成情况

### 2.1 首屏结构重构

已完成：

- 将工作台首屏调整为三栏结构
- 左侧固定为模式与模板选择区
- 中间固定为上传与预演主舞台
- 右侧固定为上传文件清单区
- 页面容器已从居中定宽调整为全屏拉宽展示

这意味着首屏的信息密度虽然提升了，但职责已经更稳定：

- 左侧负责“配置”
- 中间负责“上传与预演”
- 右侧负责“文件检查与移除”

### 2.2 上传交互收敛

已完成：

- 中间区域保留大上传框作为主入口
- 支持拖拽上传和点击上传
- 上传后中间区域不再继续堆叠配置表单
- 上传后自动切换为“表格样式预演”状态
- 预演区顶部增加“返回补充文件”按钮
- “开始处理”按钮固定在中间主舞台卡片底部
- 按钮不再跟随上传框或预演内容高度上下漂移
- 上传框内部图标、标题、按钮和标签区的纵向比例已重新校正
- 上传框在高容器中的内容分布已改为更稳定的居中节奏

当前的交互含义是：

- 用户第一次进入页面时，优先做上传
- 文件进入队列后，中心区域改为让用户理解“最终会生成怎样的表格形态”
- 如果要继续补文件，可以通过返回按钮切回上传框

### 2.3 模板与标准字段预演

已完成：

- 模板模式下，中间区域根据当前所选模板渲染虚拟表头
- 首列显式预演为“源文件”
- 标准字段模式下，中间区域改为预演标准字段结构
- 预演区采用表头 + 骨架行的虚拟表格形式，而非纯文本标签
- 预演表格容器改为占满中间卡片剩余高度
- 纵向空间优先分配给表格滚动区，提升预览行数和整体可读性

这样处理后，上传后的反馈不再只是“文件已上传”，而是“这批文件将按怎样的结构进入结果表格”。

### 2.4 文件清单侧栏

已完成：

- 将文件列表从上传组件中拆出，独立到右侧栏
- 收敛为紧凑型表格式列表，而非卡片堆叠
- 新增序号列，便于和重复提示联动理解
- 显示文件名
- 显示文件大小
- 正常文件显示只读勾选状态
- 重复文件在状态列显示红色 `×` 图标
- 重复文件在状态图标下补充“与 N 重复”或“与已有文件重复”提示
- 重复文件可直接点击状态列中的红色 `×` 删除当前文件
- 删除了旧版标题、统计、说明、文件类型与一键清空等冗余内容

这样处理后，右侧区域只保留与用户当前判断最相关的信息：

- 第几个文件
- 是什么文件
- 文件有多大
- 是否重复，以及与谁重复

这使得上传主入口和文件管理职责继续分离，同时把文件检查动作收敛到最短路径，避免右侧列表再次膨胀成另一个功能面板。

### 2.5 处理中保护

已完成：

- 处理中时，中间主舞台继续保留虚化遮罩和进度层
- 点击开始处理后，中间主舞台直接切换为进度分析层，覆盖原本的上传框或表格预演
- 处理中时禁用模式切换
- 处理中时禁用模板切换

这样可以避免任务已创建后，前端状态仍被用户继续修改，导致界面与任务实际状态错位。

补充说明：

- 任务完成后，中间主舞台不再回到上传预演，而是直接切换为结果预览表格
- 下方结果区收敛为导出下载与失败项承载区域

### 2.6 主题与样式方向

目前已完成的视觉方向调整包括：

- 使用浅色暖底而不是深色科技风
- 强化大圆角卡片与低对比边框
- 弱化旧版密集面板堆叠感
- 把上传区、模板区、文件区做出明确层次
- 继续保持结果区和导出区作为后置内容

## 5. 当前验收结论

目前已经满足以下要求：

1. 页面可按左、中、右三个区域理解。
2. 左侧可承载模板选择。
3. 中间保留上传选择框。
4. 右侧集中展示上传文件内容。
5. 文件列表以序号、文件名、大小、状态四列展示。
6. 文件列表优先给文件名留出展示空间。
7. 重复文件状态会显式展示，并提供“与 N 重复”提示。
8. 重复文件支持直接点击红色 `×` 删除。
9. 上传后中间区域会切换成模板或标准字段预演态。
10. 中间区域顶部提供返回按钮，允许继续补充文件。
11. 页面已改为更充分利用整屏宽度。
12. 处理中进度轮盘已修正为稳定居中显示，内外圈比例一致。
13. 进度面板头部已移除“处理进度”标题，仅保留状态胶囊。
14. 中间预演表格已改为固定视口高度，内容过多时在盒子内部滚动。
15. 结果区表格已按模式拆分滚动策略，模板预览和标准字段编辑不再互相干扰。
