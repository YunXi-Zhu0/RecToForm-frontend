# 本次接口优先实现总结

## 1. 本次交付结论

本次工作已经将前端从默认 Vite 示例页改造成可用的单页工作台，并按“先接口、后样式”的要求，优先完成了与后端 `http://127.0.0.1:8080` 的核心交互链路。

当前已经落地的重点不是视觉层，而是下面这条业务主链路：

1. 初始化模板列表与标准字段
2. 切换 `template` / `standard_edit` 两种模式
3. 选择并校验上传文件
4. 通过 `multipart/form-data` 创建异步任务
5. 以 2.5 秒间隔轮询任务状态
6. 任务结束后拉取 `/result`
7. 模板模式直接下载 Excel
8. 标准字段模式在前端编辑后再导出 Excel

## 2. 已完成范围

### 2.1 页面和模块结构

已经按模块化边界完成基础拆分，避免将逻辑堆进 `App.tsx`：

```text
src/
  api/
  components/
  core/
  hooks/
  pages/
  types/
  workflows/
```

页面层已切换为轻入口 + 工作台组合模式：

- `src/App.tsx`：应用轻入口
- `src/pages/workbench/workbench-page.tsx`：工作台页面装配

### 2.2 API 层

已完成以下接口封装：

- `GET /api/v1/templates`
- `GET /api/v1/templates/{template_id}`
- `GET /api/v1/standard-fields`
- `POST /api/v1/tasks`
- `GET /api/v1/tasks/{task_id}`
- `GET /api/v1/tasks/{task_id}/result`
- `POST /api/v1/exports/standard-fields`

同时保留了以下下载地址构造能力：

- `GET /api/v1/tasks/{task_id}/excel`
- `GET /api/v1/exports/standard-fields/{export_id}`

其中任务创建已严格按后端约定实现：

- 使用 `multipart/form-data`
- `config` 作为 JSON 字符串传输
- 文件字段名重复使用 `files`
- `template` 模式携带 `template_id`
- `standard_edit` 模式不携带 `template_id`

### 2.3 工作流层

已落地四个核心工作流：

- `initialize-workbench-flow.ts`
  - 并行拉取模板列表和标准字段
  - 自动选择首个模板并继续拉取模板详情
- `create-task-flow.ts`
  - 按模式组装任务创建参数
- `resolve-task-result-flow.ts`
  - 处理 `/result` 返回 `409` 的“结果尚未就绪”场景
- `export-standard-fields-flow.ts`
  - 从当前编辑态表格重新组装 `headers + rows`
  - 导出前校验表头、列数和空值约束

### 2.4 Hook 与状态模型

本次已经把工作台的核心状态收敛到 Hook 中：

- `use-workbench-state.ts`
  - 初始化
  - 模式切换
  - 模板详情加载
  - 任务提交
  - 轮询收敛
  - 结果装载
  - 标准字段导出
- `use-task-polling.ts`
  - 轮询周期为 `2500ms`
  - 当状态进入 `succeeded`、`partially_succeeded`、`failed` 时进入结果解析流程
- `use-upload-files.ts`
  - 本地文件列表维护
  - 文件数、大小、类型校验
  - 弱重复提示
  - 后端重复文件标记

当前已维护的核心状态包括：

- `mode`
- `templates`
- `selectedTemplateId`
- `templateDetail`
- `standardFields`
- `uploadFiles`
- `taskId`
- `taskStatus`
- `taskStage`
- `taskProgress`
- `taskResult`
- `failedItems`
- `editableHeaders`
- `editableRows`
- `exportId`
- `exportDownloadUrl`

### 2.5 错误处理与边界处理

本次已建立统一错误解析基础设施：

- 后端常规错误：解析 `detail`
- 重复文件错误：解析 `detail.message + detail.duplicate_files`
- `409 Conflict`：识别为结果未就绪，继续轮询

上传侧已补齐的轻量校验包括：

- 最大文件数 `50`
- 单文件最大 `10MB`
- 只允许图片与 PDF
- 本地弱重复告警

对后端重复文件错误的处理也已经符合约定：

- 不进入轮询态
- 保留当前用户已选文件
- 在对应文件项上标记重复状态

### 2.6 结果展示与导出

模板模式已支持：

- 使用 `preview_headers + preview_rows` 只读展示
- 使用 `/result` 返回的 `excel_download_url` 下载结果

标准字段模式已支持最小可用编辑能力：

- 编辑表头
- 编辑单元格
- 删除列
- 左右移动列顺序
- 调用导出接口
- 使用导出响应中的 `download_url` 下载文件
