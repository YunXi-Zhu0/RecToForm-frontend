# Phase 1：联调后端接口并实现基本页面

## 1. 阶段目标

Phase 1 的目标是将前端从默认 Vite 示例页改造成可用的单页工作台，并优先打通与后端的核心业务链路，先完成“接口可用、流程可跑、页面可展示”的基础版本。

本阶段聚焦的不是复杂视觉效果，而是下面这条主流程：

1. 初始化模板列表与标准字段
2. 切换 `template` / `standard_edit` 两种模式
3. 选择并校验上传文件
4. 通过 `multipart/form-data` 创建异步任务
5. 轮询任务状态与进度
6. 任务结束后拉取 `/result`
7. 模板模式下载 Excel
8. 标准字段模式预览并准备后续编辑导出

## 5. 已完成的实现范围

### 5.1 API 层与工作流层

已完成以下关键工作流：

- `initialize-workbench-flow.ts`
  - 并行拉取模板列表与标准字段
  - 自动选中首个模板并加载模板详情
- `create-task-flow.ts`
  - 按模式组装建任务参数
- `resolve-task-result-flow.ts`
  - 处理 `/result` 返回 `409` 的结果未就绪场景
- `export-standard-fields-flow.ts`
  - 基于当前前端表格状态重新组装 `headers + rows`

### 5.2 状态管理与 Hook

核心状态已收敛到 Hook：

- `use-workbench-state.ts`
  - 初始化
  - 模式切换
  - 模板详情加载
  - 任务提交
  - 轮询收敛
  - 结果装载
  - 导出流程
- `use-task-polling.ts`
  - 轮询间隔 `2500ms`
  - 在终态时进入结果解析
- `use-upload-files.ts`
  - 文件列表维护
  - 本地上传校验
  - 弱重复提示
  - 后端重复文件标记

### 5.3 错误处理与边界处理

已建立统一错误解析能力：

- 常规错误解析 `detail`
- 特殊解析 `detail.duplicate_files`
- `/result` 返回 `409 Conflict` 时继续轮询

上传侧已补齐的轻量校验包括：

- 最大文件数 `50`
- 单文件最大 `10MB`
- 只允许图片与 PDF
- 本地弱重复提示

### 5.4 结果预览与导出

模板模式已支持：

- 使用 `preview_headers + preview_rows` 只读展示
- 使用 `/result` 返回的 `excel_download_url` 下载结果

标准字段模式在 Phase 1 阶段已具备最小可用结果预览与导出基础：

- 获取 `standard_fields + rows`
- 渲染预览表格
- 从前端当前表格状态重新组装导出数据
- 使用导出响应中的 `download_url` 下载文件

## 6. 已确认的实现约束

### 6.1 任务创建

已确认符合后端约定：

- 使用 `multipart/form-data`
- `config` 以 JSON 字符串传输
- 文件字段名重复使用 `files`
- `template` 模式携带 `template_id`
- `standard_edit` 模式不携带 `template_id`

### 6.2 轮询与结果收敛

已确认前端实现包含：

- 轮询间隔 `2500ms`
- 终态 `succeeded`、`partially_succeeded`、`failed` 停止状态轮询
- `/result` 返回 `409` 时继续等待结果就绪

### 6.3 错误处理

已确认前端实现包含：

- 常规 `detail` 错误解析
- `detail.duplicate_files` 特殊错误解析
- 重复文件时不进入轮询，保留当前文件选择
