# Phase 3：增强稳定性与体验

## 1. 本次目标

在 Phase 2 已完成 `Handsontable` 在线编辑能力的基础上，继续补齐工作台在稳定性、可恢复性和可操作性上的增强项，覆盖以下目标：

1. 失败项过滤与聚合展示
2. 标准字段模式草稿缓存
3. 自定义导出文件名
4. 恢复原始识别值

同时修正一个在实际编辑中暴露出来的交互问题：

- 自定义导出文件名输入框在中文输入法场景下，输入一个字后会被父级重渲染打断，必须重新聚焦才能继续输入

## 2. 最终实现结论

截至 2026-03-28，本次 Phase 3 已完成以下能力落地：

- `standard_edit` 模式下支持本地草稿自动保存
- 页面初始化时若检测到草稿，可自动恢复到上次编辑中间态
- 支持从当前编辑结果一键恢复到原始识别值
- 失败项区域支持按错误原因聚合、按关键字过滤
- 导出区支持自定义导出文件名
- 导出文件名输入框已改为本地输入态，避免中文输入法被打断

这意味着标准字段模式已经不再只是“能编辑”，而是具备了继续编辑、恢复、筛选异常和稳定导出的基本工作台能力。

## 3. 关键实现方式

### 3.1 草稿缓存与恢复策略

本次新增了标准字段模式草稿存储模块，将以下内容写入浏览器本地存储：

- 当前任务结果 `task_result`
- 原始表头 `original_headers`
- 原始结果行 `original_rows`
- 当前编辑中的表头 `editable_headers`
- 当前编辑中的表格行 `editable_rows`
- 当前输入的导出文件名 `export_filename`
- 草稿保存时间 `saved_at`

存储方式特点：

- 仅对 `standard_edit` 结果生效
- 每次编辑表格或修改导出文件名后自动覆盖保存
- 若草稿结构不合法，会自动丢弃，避免污染页面状态

页面初始化时会优先检查本地草稿：

- 有草稿则直接恢复到 `standard_edit` 编辑态
- 无草稿则按原流程进入默认 `template` 模式

### 3.2 原始识别值恢复策略

为支持“回退到后端原始识别结果”，本次在工作台状态中将原始结果与编辑结果拆成两份：

- `originalEditableHeaders + originalEditableRows`
- `editableHeaders + editableRows`

这样可以在不依赖后端再次请求结果的情况下，直接把当前表格恢复为最初识别值。

恢复动作特点：

- 只在当前编辑态与原始结果存在差异时可用
- 恢复后会清空旧的导出链接，避免下载过期导出结果

### 3.3 失败项过滤与聚合展示

原来的失败项区只是一份静态列表，本次改成了更适合联调和排查的展示方式：

- 展示失败文件总数
- 展示失败原因种类数
- 展示当前筛选命中的结果数
- 支持按错误原因聚合
- 支持按文件名或错误信息关键字搜索

这样在任务部分失败时，用户可以更快地定位问题文件和问题类型，而不是逐条手工翻找。

### 3.4 自定义导出文件名与输入框修正

导出文件名能力本次分成了两个层次：

- 页面状态中维护“最终提交给导出流程的文件名”
- 输入框内部维护“当前正在输入的本地值”

这样处理的原因是：

- 如果每次键入都直接同步到父级状态，会触发页面重渲染
- 在中文输入法场景下，重渲染会打断组合输入，表现为“打一个字就失焦”

本次修正后：

- 输入框不再自动判断或补全 `.xlsx`
- 输入框允许保持为空
- 失焦时再提交当前值到页面状态
- 点击导出时直接使用输入框当前值，避免父级状态异步更新导致拿到旧值
- 若最终文件名为空，则回退到后端默认文件名

## 6. 验证结果

本次改动已完成以下验证：

- `npm run lint`
- `npm run build`

验证通过，当前仍保留一个既有告警：

- 构建产物中 `Handsontable` 相关 chunk 体积较大，`vite build` 仍会提示 chunk size warning

该告警不影响功能正确性，但后续可以考虑将数据网格相关代码做懒加载或拆分 chunk。

## 7. 当前已知限制与后续建议

虽然 Phase 3 已完成，但还可以继续增强：

- 增加草稿清空或草稿版本切换的显式 UI
- 为失败项增加“仅看当前失败原因”之外的批量重试入口
- 为恢复原始识别值增加更明确的二次确认或差异提示
- 继续处理 `Handsontable` 的包体积问题

## 8. 本次落地价值

这次改动的价值在于，`standard_edit` 模式已经具备更接近真实业务工作台的稳定性：

- 编辑过程中可以中断并恢复
- 错误信息可以更快聚合定位
- 原始识别结果可以回退
- 导出时可以保持用户期望的命名方式

这让后续继续做更复杂的编辑体验增强时，不必再先补一轮基础稳定性设施。
