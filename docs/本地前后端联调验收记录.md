## 6. 已确认的实现约束

### 6.1 任务创建

已确认符合后端约定：

- 使用 `multipart/form-data`
- `config` 以 JSON 字符串传输
- 文件字段名重复使用 `files`
- 模板模式带 `template_id`
- 标准字段模式不带 `template_id`

### 6.2 轮询与结果收敛

已确认前端实现包含：

- 轮询间隔 `2500ms`
- 终态 `succeeded`、`partially_succeeded`、`failed` 停止状态轮询
- `/result` 返回 `409` 时不报错退出，而是继续等待结果就绪

### 6.3 错误处理

已确认前端实现包含：

- 常规 `detail` 错误解析
- `detail.duplicate_files` 特殊错误解析
- 重复文件时不进入轮询，保留原文件选择

### 6.4 结果与导出

已确认前端实现包含：

- 模板模式使用 `preview_headers + preview_rows`
- 模板模式优先使用 `/result` 返回的 `excel_download_url`
- 标准字段模式从当前编辑态 `headers + rows` 发起导出
- 标准字段模式使用导出响应中的 `download_url`
