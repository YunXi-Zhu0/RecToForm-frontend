# RecToForm UI 重构上下文

## 1. 当前重构目标

当前 UI 重构的核心目标已经明确，不再是“深色科技感工作台”，而是将页面收敛为一个更轻松、更简约、更自然的单页流程。

重构必须满足以下用户已确认要求：

- 页面太复杂，需要简约
- 排版布局要合理，不冗余
- 商标 `RecToForm` 作为页头放在上方
- 页面中间部分有一个大的上传框
- 上传框支持拖拽上传
- 上传文件后，在同一块区域要求用户选择处理模式
- 处理中时虚化背景进行进度显示
- 进度显示中有轮盘
- 整体主题不要科技感，改为轻松、简约

## 2. 当前页面结构结论

页面当前应固定为以下主结构：

1. 顶部简洁品牌头部
2. 中部单一主工作区
3. 任务完成后再出现的结果后置区域

其中，中部单一主工作区承担完整前置流程：

1. 上传文件
2. 选择处理模式
3. 选择模板或查看标准字段配置
4. 启动任务
5. 处理中显示虚化遮罩和轮盘进度

这块区域不能再拆成多个分散面板。

## 3. 视觉方向结论

当前视觉方向已经切换为：

- 浅色
- 柔和
- 简洁
- 留白更足
- 类似纸面办公工具

明确不再采用：

- 深色未来感
- 数据控制台
- 网格光晕和强发光边框
- 过重的仪表盘式科技界面

## 4. 已完成的关键实现

当前代码已完成以下重构：

- 将页面首屏从多面板工作台收敛为单核心主卡片
- 将 `RecToForm` 固定为页面顶部品牌头部
- 将上传区提升为首屏唯一视觉中心
- 在上传完成后，于同一区域展开模式选择
- 模板模式下在同一区域补充模板配置
- 在任务处理中，对主区域做虚化遮罩
- 在遮罩中心展示轮盘式进度组件
- 将结果区、导出区、失败项区后置到任务完成之后
- 将整体主题由科技感深色方案切换为轻松简约的浅色方案

## 5. 当前代码锚点

继续迭代 UI 时，应优先从以下文件入手：

- `src/pages/workbench/workbench-page.tsx`
- `src/components/upload-panel/upload-panel.tsx`
- `src/components/mode-switcher/mode-switcher.tsx`
- `src/components/progress-panel/progress-panel.tsx`
- `src/components/result-table/result-table.tsx`
- `src/components/export-actions/export-actions.tsx`
- `src/components/failed-items/failed-items.tsx`
- `src/index.css`

## 6. 后续迭代约束

后续继续改 UI 时必须遵守：

- 不得把首屏重新改回多栏控制台布局
- 不得回退到深色科技感主题
- 不得把模式切换放到上传区之外的远处
- 不得让结果区在无结果时抢占首屏
- 进度显示仍应保留虚化覆盖层和轮盘

## 7. 当前重构状态

当前状态可视为：

- 页面主流程重构：已完成
- 主题切换：已完成
- 文案方向调整：已完成一轮
- 后续细节润色：可继续迭代

如果后续继续优化，应优先做：

1. 微调留白与字号节奏
2. 压缩仍偏重的结果区样式
3. 检查移动端主区域比例
4. 继续减少不必要的标签和提示语
